SKILL DRAFT — Ytel Daily Monitor ADP
======================================
PURPOSE: When this skill is invoked, Claude knows exactly what every column,
metric, and business concept means for this call center dashboard. No need to
re-explain terminology each session.

======================================
SECTION 1 — XLSX COLUMNS (what each column means)
======================================

call_date
  - The exact date and time the call was made or received
  - Format: 2026-06-18 09:33:34
  - Used to filter all calls to a specific analysis date

direction
  - "inbound" = client called us
  - "outbound" = we called the client

phone_number / phone_number_dialed
  - The client's phone number
  - Used as the unique identifier for a client across all calls

length_in_sec
  - How long the call lasted in seconds
  - 0–30s = dead call (nobody really talked)
  - 30–120s = short call
  - 120–300s = 2 to 5 minutes
  - 300s+ = real conversation started (>5 min is minimum for enrollment opportunity)
  - 900s+ = strong conversation (>15 min)
  - 1800s+ = very deep conversation (>30 min)

status (dispo code)
  - SALE = agent closed a sale / made a deal
  - CC = Current Client (existing enrolled client)
  - DNC = Do Not Call — this number should not be dialed
  - DROP = call dropped before agent connected
  - TIMEOT = call timed out, agent did not pick up
  - N = No Answer from client
  - CLtrns = Call Transfer (client was transferred to another agent)
  - [ADD YOUR OTHER DISPO CODES AND WHAT THEY MEAN]

status_name
  - Human-readable version of the status code
  - Example: TIMEOT → "No Answer", SALE → "Sale Made", DNC → "Do Not Call"

full_name
  - The agent's display name
  - Used for all per-agent stats and enrollment credit

user
  - Agent's numeric ID in Ytel
  - VDCL and VDAD are system/dialer rows — excluded from agent performance table

campaign_id
  - Which campaign or queue the call came from
  - Examples: 1000 (main outbound), AGENTDIRECT (client called agent directly),
    TransferK (transfer queue), PLR (inbound marketing)
  - [ADD YOUR CAMPAIGN IDs AND WHAT EACH ONE MEANS IN YOUR BUSINESS]

source_id
  - How the call was routed
  - VDCL = auto-dialer system drop (not a real agent conversation)
  - AGENTDIRECT = client called in directly to a specific agent
  - MANUAL = agent manually dialed

user_group
  - The team the agent belongs to
  - Example: Team_1

CRM Status
  - The lead's current status in the CRM at time of call
  - Example: Waiting_For_First_Payment = enrolled but hasn't paid yet
  - [ADD ALL YOUR CRM STATUSES AND WHAT THEY MEAN]

Enrolled Debt
  - The dollar amount of debt the client enrolled with
  - Example: $22,455.00
  - Only meaningful on rows where Cordoba Enrolled Date is populated

Cordoba Enrolled Date
  - The date the client was officially enrolled in the program
  - If this matches the analysis date → this phone counts as an enrollment today
  - This is the single most important column for measuring sales performance

Assigned To
  - The agent who should receive credit for the enrollment
  - Overrides the automatic last-call credit logic
  - If blank → credit goes to the agent with the last call to that phone that day

Last Credit Pulled Date
  - Date the client's credit was last pulled — NOT the enrollment date
  - Do not confuse with Cordoba Enrolled Date

SECTION 2 — KEY METRICS AND WHAT THEY MEAN
======================================

Total Calls
  - Every single call row for the day, all dispositions included
  - Includes dead calls, drops, timeouts, real conversations — everything

Unique Calls (Unique #s)
  - Number of distinct phone numbers contacted
  - If an agent called the same number 5 times, it still counts as 1 unique
  - Better measure of reach than total calls

Short% (Dead ≤30s)
  - Percentage of calls that lasted 30 seconds or less
  - These calls had no real conversation — wasted attempts
  - Above 55% for an agent = coaching flag
  - High short% across the board = list quality problem or agents hanging up early

>2 Min %
  - Percentage of calls that lasted more than 2 minutes
  - For OPENERS this is the key metric — did they get the client talking?
  - Green if ≥50%, red if below
  - Not shown for closers (different metric applies to them)

>5 Min
  - Calls over 5 minutes
  - Minimum threshold for a real sales conversation
  - Enrollment conversion rate is calculated against this number (enrolled ÷ >5min calls)

>15 Min / >30 Min / >45 Min
  - Deep conversations
  - Counted by UNIQUE phone number (best call per phone), not raw call count
  - [ADD: what does a 30min+ call mean in your business — is it always good?]

Conv% (ENR/>5M)
  - Conversion rate = enrollments ÷ calls over 5 minutes
  - The core performance metric for closers
  - [ADD: what is a good Conv% in your business?]

Avg Talk
  - Average call duration across all calls for that agent
  - Total seconds ÷ total calls
  - [ADD: what avg talk time is healthy for openers vs closers?]

Total Talk
  - Sum of all call durations for that agent that day
  - Shows how much time was actually spent on the phone

Enrolled
  - Number of unique phone numbers that agent got credit for enrolling today
  - Uses Assigned To if populated, otherwise last-call timestamp wins

Debt $
  - Total dollar value of debt enrolled by that agent today
  - Sum of Enrolled Debt for all phones credited to that agent

SECTION 4 — ENROLLMENT RULES
======================================

1. A phone number is counted as enrolled only if its Cordoba Enrolled Date
   matches the date being analyzed. Enrollment on a past date does not count
   toward today's numbers.

2. One enrollment per unique phone number per day — no matter how many times
   that phone was called or how many rows have the enrolled date filled in.

3. Credit goes to the agent in the "Assigned To" column first.
   If that column is blank, credit goes to the agent who made the
   LAST call to that phone number on that day (by timestamp).

4. Debt amount comes from the Enrolled Debt column on the credited row.

======================================
SECTION 5 — HOURLY BREAKDOWN RULES
======================================

Unique #s per hour:
  - A phone number is counted in the hour of its FIRST call of the day
  - Even if the agent called that number 10 times across the day,
    it only shows up once, in the first hour it was dialed

Enrolled per hour:
  - The enrollment is credited to the hour of the agent's LAST call
    to that enrolled phone number
  - Example: agent first calls at 9am, calls back at 10am and closes —
    the enrollment shows in the 10am bucket, the unique shows in 9am bucket

======================================
SECTION 6 — MISSED CALLBACKS
======================================

What it tracks:
  - Inbound TIMEOT calls — the client called us and nobody picked up
  - These are the most important missed calls because the CLIENT reached out

A phone is considered resolved (not flagged) if:
  - Any call (inbound OR outbound, by any agent) happened AFTER the last timeout
  - Even if the client called back themselves and got through, that counts

A phone is flagged only if:
  - Zero follow-up of any kind occurred after the last timeout

Split into two groups:
  - Enrolled Clients = phone has a Cordoba Enrolled Date (existing client ignored)
  - Other Calls = no enrolled date (lead or unknown caller ignored)

Why this matters:
  - Enrolled clients who called in and got no answer = serious service failure
